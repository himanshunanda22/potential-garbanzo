Section 01

Markov Decision Process

A MDP provides the mathematical backbone for reasoning over attacker sessions. Rather than classifying each request in isolation, the system models the entire session as a trajectory through state space and selects actions that maximise long-run cumulative reward.

MDP tuple Definition
The system is fully described by five components:
\(\mathcal{M} = \langle \mathcal{S},\, \mathcal{A},\, \mathcal{T},\, \mathcal{R},\, \gamma \rangle\)
\(\mathcal{S}\)State space — all possible session trajectory embeddings (104-dimensional vectors) \(\mathcal{A}\)Action space — {DECEIVE, DEEPEN_TRAP, ESCALATE, HARD_EXIT} \(\mathcal{T}\)Transition function — \(P(s' \mid s, a)\), probability of next state given current state and action \(\mathcal{R}\)Reward function — \(r: \mathcal{S} \times \mathcal{A} \to \mathbb{R}\), scalar signal after each action \(\gamma\)Discount factor — \(\gamma = 0.95\), controls how much future rewards are valued
State representation State space
Each attacker session is encoded as a fixed-size state vector by concatenating the last \(W = 8\) feature vectors. This projects a finite observation window into an infinite-horizon value estimate.
\(s_t = \bigl[\phi(x_{t-W+1}),\; \phi(x_{t-W+2}),\; \ldots,\; \phi(x_t)\bigr] \in \mathbb{R}^{W \times d}\)
\(\phi(x_t)\)Feature vector for request \(t\) — 13-dimensional: SQL score, XSS score, traversal score, prompt score, scanner score, body length, auth header, header count, path depth, query params, method score, content-type mismatch, referer \(W\)Session window size = 8 (zero-padded for early requests) \(d\)Feature dimension = 13, giving total state dim = \(8 \times 13 = 104\)
Optimal policy Objective
The agent learns a policy \(\pi^*\) that maximises expected discounted cumulative reward over an infinite horizon:
\(\pi^* = \arg\max_\pi\; \mathbb{E}_\pi \!\left[ \sum_{k=0}^{\infty} \gamma^k\, r_{t+k} \;\middle|\; s_t \right]\)
\(\gamma^k\)Exponential discounting — rewards 10 steps ahead are worth \(0.95^{10} \approx 0.60\) of immediate rewards \(\pi(a \mid s)\)Policy — probability of taking action \(a\) in state \(s\)
Section 02

ThreatNet Classifier

A feed-forward neural network that performs dual classification: detecting whether a request is a threat, and identifying the threat type. Its output feeds directly into the MDP state.

Network architecture Forward pass
Three hidden layers with ReLU activations and dropout for regularisation. Two output heads share the same trunk:
\(h_1 = \text{ReLU}(W_1 \phi + b_1) \in \mathbb{R}^{64}\)
\(h_2 = \text{ReLU}(W_2\, \text{Drop}(h_1) + b_2) \in \mathbb{R}^{64}\)
\(h_3 = \text{ReLU}(W_3\, \text{Drop}(h_2) + b_3) \in \mathbb{R}^{32}\)
\(\hat{p} = \sigma(w_p^\top h_3) \in [0,1], \qquad \hat{y} = \text{softmax}(W_y h_3) \in \mathbb{R}^6\)
\(\hat{p}\)Threat probability — request flagged if \(\hat{p} > 0.65\) \(\hat{y}\)Threat type logits over 6 classes: SQL injection, XSS, path traversal, prompt injection, scanner, unknown \(\sigma\)Sigmoid activation: \(\sigma(z) = \frac{1}{1+e^{-z}}\)
Training loss Optimisation
The classifier optimises a combined loss — binary cross-entropy for the threat head, cross-entropy for the type head, with a weight of 0.5 on the type loss:
\(\mathcal{L}_\text{clf} = \underbrace{-\bigl[y\log\hat{p} + (1-y)\log(1-\hat{p})\bigr]}_{\text{BCE threat detection}} +\; 0.5\underbrace{\sum_{c=1}^{6} y_c \log \hat{y}_c}_{\text{CE type classification}}\)
Section 03

Deep Q-Learning & Double DQN

The policy engine uses a Deep Q-Network (DQN) to approximate the action-value function. The Double DQN variant decouples action selection from action evaluation, reducing the overestimation bias of vanilla DQN.

Bellman optimality equation Core equation
The optimal Q-function satisfies the Bellman equation — the value of taking action \(a\) in state \(s\) equals the immediate reward plus the discounted value of the best next action:
\(Q^*(s, a) = \mathbb{E}\!\left[ r + \gamma \max_{a'} Q^*(s', a') \;\middle|\; s, a \right]\)
\(Q^*(s,a)\)Optimal action-value: expected return starting from \(s\), taking \(a\), then acting optimally \(r\)Immediate reward from the reward shaping function \(\gamma = 0.95\)Discount — one step into the future is worth 95% of the present
Double DQN target Training target
Standard DQN uses the same network to select and evaluate the next action, causing upward bias. Double DQN uses two separate networks — the online network selects the action, the target network evaluates it:
\(y_t = r_t + \gamma\; Q_{\theta^-}\!\!\left(s_{t+1},\; \arg\max_{a'} Q_\theta(s_{t+1}, a')\right)\)
\(Q_\theta\)Online network — parameters \(\theta\) updated every step via gradient descent \(Q_{\theta^-}\)Target network — parameters \(\theta^-\) copied from \(\theta\) every 50 steps, held fixed between copies \(y_t\)TD target — the "ground truth" Q-value we train the online network to predict
Huber loss (Smooth L1) Loss function
More robust than MSE for Q-learning — behaves like L2 for small errors and L1 for large ones, preventing exploding gradients from outlier transitions:
\(\mathcal{L}(\delta) = \begin{cases} \tfrac{1}{2}\delta^2 & \text{if } |\delta| \leq 1 \\ |\delta| - \tfrac{1}{2} & \text{otherwise} \end{cases} \quad \text{where } \delta = y_t - Q_\theta(s_t, a_t)\)
Epsilon-greedy exploration Exploration policy
The agent explores randomly early in training and gradually exploits its learned policy. Epsilon decays exponentially:
\(\varepsilon_t = \varepsilon_\text{end} + (\varepsilon_\text{start} - \varepsilon_\text{end})\cdot e^{-t / \tau}\)
\(\varepsilon_\text{start} = 1.0,\quad \varepsilon_\text{end} = 0.05,\quad \tau = 500\)
The action is selected as:
\(a_t = \begin{cases} \text{random } a \in \mathcal{A} & \text{with prob. } \varepsilon_t \\ \arg\max_{a} Q_\theta(s_t, a) & \text{with prob. } 1-\varepsilon_t \end{cases}\)

Section 05

Reward Shaping

The reward function is handcrafted to encode desired behaviour — the agent should string attackers along as long as possible, escalate high-confidence threats, and avoid premature exits that cut off information gathering.

Reward function \(\mathcal{R}(s, a)\) Full definition
Each action accumulates reward components based on the current state:
\(r_\text{DECEIVE} = \mathbf{1}[\text{captured}] + 0.3 \cdot \min(d, 10)\)
\(r_\text{DEEPEN} = 1.5\cdot\mathbf{1}[\text{captured}] + 0.5\cdot\min(d,8) - 0.5\cdot\mathbf{1}[\text{repeat}]\)
\(r_\text{ESCALATE} = 3.0\cdot\mathbf{1}[\hat{p} > 0.85] + 0.5\cdot\mathbf{1}[\hat{p} \leq 0.85] - 1.0\cdot\mathbf{1}[\hat{p} < 0.70]\)
\(r_\text{EXIT} = 2.0\cdot\mathbf{1}[\hat{p} > 0.90 \vee \text{repeat}] - 2.0\cdot\mathbf{1}[\hat{p} \leq 0.90] - 0.5d\)
\(d\)Session depth — number of requests captured so far in this session \(\hat{p}\)Threat probability from ThreatNet classifier \(\mathbf{1}[\cdot]\)Indicator function — equals 1 if condition true, 0 otherwise \(\text{repeat}\)True if last 3 payloads share identical hash (attacker probing detection)

Section 06

System Architecture

The full system pipeline — from raw HTTP request through feature extraction, classification, MDP decision, deception execution, and learning update.

Request lifecycle
Inference path
Learning path
New in v2
1
HTTP request received
FastAPI middleware intercepts every request before it reaches the protected route. Dashboard and internal API routes are excluded from inspection.
2
Feature extraction — \(\phi(x_t) \in \mathbb{R}^{13}\)
13 scalar features extracted from request headers, URL, method and body: pattern match scores for SQL injection, XSS, path traversal, prompt injection and scanner signatures; plus structural signals like body length, auth header presence and referer.
\(\phi(x_t) = [\text{sql\_score},\; \text{xss\_score},\; \ldots,\; \text{referer\_missing}]\)
3
ThreatNet inference — \(\hat{p} = \sigma(w_p^\top h_3)\)
The 3-layer classifier produces a threat probability and threat type. If \(\hat{p} \leq 0.65\), the request is passed through to the real handler. If \(\hat{p} > 0.65\), the MDP pipeline activates.
4
Session state update — \(s_t \in \mathbb{R}^{104}\) NEW v2
The SessionTracker appends \(\phi(x_t)\) to the rolling session window. The flattened window becomes the MDP state. Payload hashes are logged to detect repetition.
\(s_t = [\phi_{t-7},\; \phi_{t-6},\; \ldots,\; \phi_t] \in \mathbb{R}^{8 \times 13}\)
5
Policy decision — \(a_t = \arg\max_a Q_\theta(s_t, a)\) NEW v2
The online Q-network evaluates all four actions. The agent selects the greedy action (or explores randomly if \(\varepsilon_t\) triggers). Hard overrides apply: scanners with score > 0.90 always trigger ESCALATE.
6
Deception execution
DECEIVE returns a standard fake 200 response. DEEPEN_TRAP returns a richer payload (fake credentials, directory listings, config data). ESCALATE logs with CRITICAL severity. HARD_EXIT returns 503 and clears session state.
7
Footprint capture
Full fingerprint stored to SQLite: SHA-256 hash of IP + user-agent + payload, session depth, threat score, Q-values for all actions, feature vector, response time, and all non-sensitive headers.
8
Online learning — Bellman update
The transition \((s_t, a_t, r_t, s_{t+1}, \text{done})\) is pushed to the replay buffer. When buffer size ≥ 64, a random minibatch is sampled and a Double DQN update step is executed. Target network syncs every 50 steps.
\(\theta \leftarrow \theta - \alpha \nabla_\theta \mathcal{L}(\delta)\)
Section 07

Training Pipeline

Training happens in two phases: offline pre-training on synthetic data to give the policy a warm start, followed by continuous online learning on live traffic.

Phase 1 — Classifier pre-training Offline
ThreatNet trained for 40 epochs on 6,000 synthetic samples (40% legitimate, 60% attack across 5 types). Adam optimiser, lr = 1×10⁻³, batch size 64.
\(\theta_\text{clf}^* = \arg\min_\theta \frac{1}{N}\sum_{i=1}^N \mathcal{L}_\text{clf}(\phi(x_i), y_i;\, \theta)\)
Phase 2 — MDP pre-training Offline
3,000 synthetic attack episodes (3–12 requests each) generated across all 5 attack profiles. Heuristic-optimal actions with 20% exploration noise used to pre-fill the replay buffer before live deployment.
\(|\mathcal{D}| \approx 3000 \times 7.5 = 22{,}500 \text{ transitions}\)
Phase 3 — Online learning Live
Every real attacker request generates one transition. Updates fire whenever the buffer holds ≥ 64 samples. The model checkpoint saves every 200 steps and hot-reloads on gateway restart.
\(\theta^- \leftarrow \theta \quad \text{every } N_\text{sync} = 50 \text{ steps}\)
\(\alpha\)Learning rate = 5×10⁻⁴ (Adam optimiser with gradient clipping at 1.0) \(|\mathcal{D}|_\text{max}\)Replay buffer capacity = 10,000 transitions (FIFO eviction)
